c++梳理（四）

viat · 2021-4-18 · 次阅读

初始化列表

只有构造函数可以使用这种初始化列表语法
在构造函数中，我们仅仅想要给类的成员变量赋初值，就可以用到初始化列表；
写在类的构造函数的参数列表后面写上要用哪个参数，来初始化哪个变量；
对于类类型来说，最好使用初始化列表,使用初始化列表少了一次调用默认构造函数的过程，这对于数据密集型的类来说，是非常高效的

类名（参数列表）：成员变量（常量或者参数）{
    //实现
}
clock(int s, int m, int h):news(s),newm(m),newh(h){
//初始化列表要写在类的声明中；
//如果还需要构造函数干别的事情，在函数体内自行实现；
}

如果初始化的数据成员是对象（继承时调用基类的构造函数），需要初始化ｃｏｎｓｔ修饰的类成员，或者需要初始化引用成员数据，必须使用初始化列表来初始化。

###哪些成员变量必须在构造函数的初始化列表中初始化
常量成员，因为常量（const）只能初始化不能赋值，所以必须放在初始化列表里面
引用类型，引用(reference)必须在定义的时候初始化，并且不能重新赋值，所以也要写在初始化列表里面
没有默认构造函数的类类型，因为使用初始化列表可以不必调用默认构造函数来初始化，而是直接调用拷贝构造函数初始化
类的ｃｏｎｓｔ成员只能使用初始化列表

析构函数

没有返回值　，也没有任何参数，　但可以是虚函数；

class A{
    ~A(){}//析构函数
};

命名空间

标识符的四种作用域：

  1. 函数原型作用域:
    在函数原型声明时形式参数的作用范围就是函数原型作用域；
    void setTime(int newS,int newM,int newH){
//我们可以看到三个形参（标识符）的作用域仅仅局限在这个个函数体内
//其他地方都不能用到这三个标识符；他们的作用域就是函数原型作用域。
}
  2. 局部作用域（块作用域）
    主要指同一函数体内，使用大括号分割的不同代码块，例如选择结构和循环结构就是局部作用域。
    对于函数体内声明的变量，其作用域从声明处开始，到声明所在的块结束，所谓块，就是大括号括起来的一段程序。
    具有局部作用域的变量也叫局部变量；
  3. 类作用域
    类作用域可以看做是一组有名字的成员的集合，某一类Class的成员member具有类作用域；
    对于member的访问，有如下几种方式；
    通过表达式object.member或者Class::member来访问（object是Class的一个实例对象）;Class::member适用于类的静态成员；
    通过ptr->member(其中ptr是指向Class的一个实例对象的指针)
  4. 命名空间（namespace）作用域
    namespace 命名空间{
　　　命名空间内的各种声明（函数声明，类声明）
}
    一个命名空间自身确定了一个作用域，凡是在该空间声明的，不属于前面提到的各个作用域的标识符（其他三个作用域），都属于这个命名空间。
    如果需要引用其他命名空间的每个标识符，可以用命名空间::标识符名
    namespace NS{
 class Class{
 //类成员
 }
}
NS::Class object;//声明一个定义在命名空间ＮＳ中类class的实例对象object;
    在标识符中总使用这样的命名空间会显得冗长，所以在程序开头可以使用using语句，如下：
  5. using 命名空间::标识符名；
    把这个命名空间中的某一标识符暴露在当前作用域，所以我们就可以使用该标识符；
  6. using namespace 命名空间；
    直接引用整个命名空间，然后可以使用该命名空间内的所有标识符；

    命名空间可嵌套

    namespace Outer{
 namespace Inner{
 class Class{
 };
 }
}
Outer::Inner::Class;

    using namespace std的使用

    这种方法可以在自己写函数，ａｃｍ中可以使用，而在大型工程中不建议使用，会造成命名空间污染；
