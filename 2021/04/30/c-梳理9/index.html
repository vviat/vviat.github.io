c++梳理（八）

viat · 2021-4-30 · 次阅读

c++面向对象

把生活中的一些方法转化为计算机语言
个体（对象）可以抽象为类
类可以实例化对象 ： 类名 + 变量名 需要调用拷贝构造函数
调用构造函数->给实例一些初始化性质
对于单个对象 有方法 访问权限 直接调用 间接调用
单个对象->继承->（已经写好的代码 和你有关系的人直接拿来用）
class A：public B 父子同名成员 父子兼容 父子+同名

模板

程序 = 数据结构+算法

#宏与函数
宏优点代码复用
缺点没有类型检查
函数优点有类型检查
缺点需要重写大量代码
模板 template
函数模板好处：代码复用 做检查，显示指定类型 也可以自动推导
是c++泛型编程之一

#多个类型的模板函数
func(a,b);
返回值类型必须显性说明
1如果函数重载和函数模板发生 默认调用函数类型明确的那个
2根据程序优先关系优先考虑普通函数。
3如果模板匹配更好优先选模板
4可以通过<返回值>，只匹配模板

类模板特化

template <typename T1 ,typename T2>
class text {
    public : 
    void add(T1 a, T2 b) {
        cout<< a + b<<endl;
    }
}

类模板实现和声明必须在一个文件中 就不会报错了

重载

如果一个作用域内几个函数名字相同但是参数列表不同

单例模式

全程只使用一次
